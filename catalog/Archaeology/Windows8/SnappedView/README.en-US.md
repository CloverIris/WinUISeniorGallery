# Snapped View (Windows 8 Design Archaeology)

This research item studies Windows 8 app snapping: tasks occupy adjacent regions instead of forcing window switching. It is historical input for `windowing.dock-layout-preview`, not an owner of window-management API. Status is `proposed` / `lab` / `P2`, license review `pending`; Gallery route `/archaeology/windows-8/snapped-view` shows structure, constraints, and modern mapping only.

Read `SPEC` for historical regions/states, `DESIGN` for retained/rejected choices, `INTEGRATION` for the host-window boundary, `ACCEPTANCE` for exhibit verification, and `SOURCES` for evidence and the asset ban. No implementation is allowed before an independent modern-owner specification.
