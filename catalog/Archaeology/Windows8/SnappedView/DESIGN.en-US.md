# Snapped View Design Archaeology and Modernization

Retain direct manipulation—preview spatial outcome before commit—and the insight that compact regions need a different information density. Reject treating a narrow region as an infinitely shrinkable full page, one fixed ratio, and system-selected task layout. Modern design must show readable previews, a minimum-content policy, and a cancellation exit.

Narrow windows use a vertical comparison; regular/wide windows use side-by-side. At 100/150/200% DPI, text scaling, Chinese/English, and RTL, use logical start/end rather than left/right semantics. Light/Dark/High Contrast preserve region and focus contrast; Reduced Motion changes state immediately. Keyboard traverses primary, secondary, explanation, and return; mouse/touch drag is simulation only; Narrator reports region role and layout.
