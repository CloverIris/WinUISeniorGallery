# Zune Big Typography Integration

## Modern Owner and Dependency

The sole modern owner is controls.big-title (BigTitle). The exhibit configures its demo only through registered public owner surfaces; stable modules never reference Archaeology.

## Gallery Data and Platform

Prototype, DNA, and differences come from local Markdown; demo uses anonymous deterministic fake data. It accesses no historical service, account, media library, or system Shell. Async assets are cancellable and carry license metadata.

## Input, Window, and Lifecycle

Demo belongs to current Gallery XamlRoot and creates/migrates no Window. Title is noninteractive and out of Tab by default; if bound to navigation it gets button semantics/focus and no invisible hit area. Unload cancels animation, delay, and Provider, ignores late results, and restores navigation focus.

## Degradation and Resources

When owner/effect is unavailable, retain text diagram, input matrix, and Sources. Archaeology never duplicates BigTitle style keys/API; historical preset exists only in Gallery configuration.

## Copyright and Privacy

license_review remains pending; every demo asset records author, license, and provenance. Upload no user media/history and collect no exhibit interaction telemetry.
