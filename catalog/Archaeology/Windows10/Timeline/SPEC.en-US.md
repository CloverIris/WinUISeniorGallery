# Timeline Specification

## Additional Given / When / Then
- Given historic/modern side by side, state/input changes make structure, focus, and removals traceable.
- Given narrow, RTL, High Contrast, or Reduced Motion, the task remains possible and Narrator gets name/role/state/position.
- Given unapproved sources/license, show original placeholders only; add no API and do not enter review/stable.

## Design DNA

Chronology, activity thumbnails, search, resume, and a unified cross-source model.

## Modern boundary

Only an app-owned ActivityTimeline concept may be demonstrated; it must not read system activity history, impersonate Task View, claim Graph resumption, or collect user documents.

## Non-goals

Do not invent history for unshipped work or describe a Gallery simulation as Windows Shell, a system service, or stable API.
