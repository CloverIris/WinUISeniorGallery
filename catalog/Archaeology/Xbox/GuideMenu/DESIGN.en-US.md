# Guide Menu Design

## Additional Given / When / Then
- Given historic/modern side by side, state/input changes make structure, focus, and removals traceable.
- Given narrow, RTL, High Contrast, or Reduced Motion, the task remains possible and Narrator gets name/role/state/position.
- Given unapproved sources/license, show original placeholders only; add no API and do not enter review/stable.

## Preserve

Preserve: Always reachable, contextual overlay, stable top-level destinations, predictable hierarchical back, controller-first vertical focus, and bumper shortcuts.

## Modernize

Reconstruct as an in-app Guide-style rail for immersive media or 10-foot apps. It carries app commands only and never replaces system navigation or intercepts reserved buttons. Use current Fluent ThemeResources, corners, typography, and system icons; historical proportions and motion are research references, not pixel copies.

## Responsive Behavior and Input

D-pad/stick navigates vertically, bumpers switch sections, A enters, and B returns; mouse, keyboard, and touch have equivalent paths. Narrow windows fall back to one column or a full-screen layer; large screens cap content width and retain 10-foot legibility.

## Motion and Accessibility

Motion conveys entry, exit, and state change only; Reduced Motion switches instantly. High Contrast does not rely on transparency, screen readers receive role, state, and next action, and focus paths are reversible.
